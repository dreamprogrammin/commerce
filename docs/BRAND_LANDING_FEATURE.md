# Brand Landing Pages (3-й уровень навигации)

## Описание

На страницах категорий второго уровня (например, `/catalog/boys/constructors`) реализован динамический вывод брендов как **третьего уровня навигации**. При клике на бренд URL обновляется на `/catalog/boys/constructors?brand=lego`, при этом:

- Автоматически фильтруются товары по выбранному бренду
- Динамически обновляется H1, Title и Description
- Страница остаётся индексируемой для поисковых систем
- Генерируется SEO-текст (из БД или автоматически)

## Как это выглядит

### До (категория без бренда)

```
URL: /catalog/boys/constructors

H1: Конструкторы для мальчиков
Title: Конструкторы для мальчиков купить в интернет-магазине Ухтышка Казахстан
Description: Конструкторы для мальчиков. ⚡ по цене от 2 500 ₸ до 45 000 ₸...

[Товары всех брендов]

--- Внизу страницы ---
[Блок "Бренды в категории «Конструкторы для мальчиков»"]
  [LEGO] [Mega Bloks] [Wange] [SluBan] ...
```

### После (категория + бренд)

```
URL: /catalog/boys/constructors?brand=lego

Breadcrumbs: Главная > Мальчикам > Конструкторы > LEGO
H1: Конструкторы для мальчиков LEGO в Алматы
Title: Купить Конструкторы для мальчиков LEGO в Алматы | Ухтышка
Description: В каталоге Ухтышка вы можете купить конструкторы для мальчиков LEGO.
             Большой выбор, гарантия оригинала, доставка по Алматы.

[Только товары LEGO]

--- Внизу страницы ---
[SEO-текст для связки LEGO + Конструкторы (из БД или автогенерация)]
[Блок "Бренды в категории" с подсветкой LEGO]
```

### Файлы созданы

| Файл | Описание |
|------|----------|
| `pages/admin/brand-seo.vue` | Админ-панель для управления SEO-текстами бренд+категория |

### База данных

**Новая таблица `category_brand_seo`:**

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | UUID | Primary key |
| `category_id` | UUID FK | Ссылка на категорию |
| `brand_id` | UUID FK | Ссылка на бренд |
| `seo_h1` | TEXT | Кастомный H1 (nullable) |
| `seo_title` | TEXT | Кастомный meta title (nullable) |
| `seo_description` | TEXT | Кастомный meta description (nullable) |
| `seo_text` | TEXT | HTML SEO-текст внизу страницы (nullable) |
| Unique constraint | | `(category_id, brand_id)` |

**Новая RPC `get_category_brand_seo(p_category_slug, p_brand_slug)`:**
- Возвращает SEO-данные для связки категория+бренд по слагам
- Используется на фронтенде для загрузки кастомного SEO

**RLS:**
- Публичный доступ на чтение (для SSR)
- Запись только для admin

## SEO стратегия

### Приоритет SEO-данных

**H1:**
1. `category_brand_seo.seo_h1` (из БД)
2. Автогенерация: `{Категория} {Бренд} в Алматы`
3. Fallback: обычный H1 категории

**Title:**
1. `category_brand_seo.seo_title` (из БД)
2. Автогенерация: `Купить {Категория} {Бренд} в Алматы | Ухтышка`
3. Fallback: обычный title категории

**Description:**
1. `category_brand_seo.seo_description` (из БД)
2. Автогенерация: `В каталоге Ухтышка вы можете купить {категория} {Бренд}...`

**SEO-текст (внизу страницы):**
1. `category_brand_seo.seo_text` (из БД, HTML)
2. Автогенерация с h2 и описанием

### Хлебные крошки (Breadcrumbs)

При активном бренде хлебные крошки расширяются — бренд добавляется как последний элемент:

| Состояние | Хлебные крошки |
|-----------|---------------|
| Без бренда | Главная > Мальчикам > Конструкторы |
| С брендом | Главная > Мальчикам > Конструкторы > **LEGO** |

- Все промежуточные крошки — кликабельные ссылки (с `href`)
- Бренд (последний элемент) — без ссылки (текущая страница)
- **Schema.org BreadcrumbList** автоматически включает бренд — Google видит иерархию «категория → бренд»
- Пример JSON-LD:
```json
{
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "position": 1, "name": "Мальчикам", "item": "https://uhti.kz/catalog/boys" },
    { "position": 2, "name": "Конструкторы", "item": "https://uhti.kz/catalog/boys/constructors" },
    { "position": 3, "name": "LEGO" }
  ]
}
```

### Индексация

- **Brand Landing (`?brand=slug`)**: `index, follow` — страницы индексируются
- **Обычные фильтры (`?brands=uuid`)**: `noindex, follow` — не индексируются
- **Canonical URL**: включает `?brand=slug` для brand landing

### Целевые запросы

| Запрос | URL |
|--------|-----|
| Конструкторы LEGO для мальчиков | `/catalog/boys/constructors?brand=lego` |
| Куклы Barbie для девочек | `/catalog/girls/dolls?brand=barbie` |
| Настольные игры Hasbro | `/catalog/board-games?brand=hasbro` |

## Кэширование

- **TanStack Query**: `queryKey` содержит `brandIds`, поэтому каждая комбинация категория+бренд кэшируется отдельно (stale: 2 мин, gc: 10 мин)
- **Pinia Store Cache**: бренды по категории кэшируются в `brandsByCategory[slug]` до закрытия вкладки
- **SEO данные**: загружаются один раз при `loadFilterData()`, обновляются при смене категории/бренда

## Админ-панель

Новая страница `/admin/brand-seo`:

- Таблица всех записей с поиском
- Создание/редактирование связки бренд+категория
- Превью URL
- Поля: H1, Title, Description, SEO-текст (HTML)
- Если поля пустые — используется автогенерация

## Дальнейшее развитие

1. **Sitemap**: добавить brand landing URL в `/api/sitemap-routes` для ускорения индексации
2. **Internal linking**: добавить ссылки на brand landing в карточки товаров
3. **Schema.org**: расширить structured data для brand landing (BrandPage)
4. **SEO 2.0 — ЧПУ URL**: когда бизнес вырастет, можно перейти на `/catalog/boys/constructors/lego` вместо `?brand=slug`. Потребует отдельного рефакторинга роутинга, но даст ещё больший SEO-буст. Пока оставляем query-параметр — он уже индексируется Google и не ломает текущую архитектуру.
