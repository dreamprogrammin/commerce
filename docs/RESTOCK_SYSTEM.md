# Система автоматических закупок

## Как работает

Система **автоматически** формирует список товаров к закупке. Никаких ручных действий для добавления товара в список не нужно.

### Логика

```
Если stock_quantity ≤ min_stock_level → товар попадает в список "К закупке"
```

- `stock_quantity` — текущий остаток на складе
- `min_stock_level` — минимальный порог (по умолчанию **2 шт**)
- `restock_quantity` — сколько нужно дозаказать (по умолчанию **5 шт**)

### Пример

| Товар | Остаток | Мин. порог | Дозаказать | В списке? |
|-------|---------|------------|------------|-----------|
| Кукла Барби | 1 | 2 | 10 | Да (1 ≤ 2) |
| Лего Сити | 5 | 2 | 5 | Нет (5 > 2) |
| Машинка RC | 0 | 3 | 8 | Да (0 ≤ 3) |
| Пазл 1000 | 2 | 2 | 5 | Да (2 ≤ 2) |

## Группировка по поставщикам

Список автоматически группируется по поставщику товара (`products.supplier_id → suppliers`).

Каждая группа показывает:
- **Название поставщика**
- **Контактное лицо**
- **Телефон** (кликабельный — можно сразу позвонить)
- **Адрес / точка на рынке**
- **Список товаров** с остатками и количеством к дозаказу

Товары без назначенного поставщика попадают в группу **"Без поставщика"** — там можно назначить поставщика прямо со страницы закупок через dropdown.

## Настройка параметров

Параметры `min_stock_level` и `restock_quantity` настраиваются для каждого товара индивидуально:

**Где**: Админка → Товары → Редактирование товара → блок "Склад"

| Поле | Описание | По умолчанию |
|------|----------|-------------|
| Остаток (шт) | Текущее количество на складе | — |
| Мин. остаток | При каком остатке товар попадёт в закупки | 2 |
| Дозаказать (шт) | Сколько штук заказать у поставщика | 5 |

## Привязка поставщика к товару

Есть **два способа** назначить поставщика товару:

### 1. Из карточки товара
Админка → Товары → Редактирование → поле "Поставщик" (dropdown)

### 2. Со страницы закупок (быстрый способ)
Админка → К закупке → в группе "Без поставщика" → dropdown "Назначить поставщика" рядом с товаром

После назначения товар автоматически перемещается в группу этого поставщика.

## Управление поставщиками

**Страница**: Админка → Поставщики (`/admin/suppliers`)

Для каждого поставщика хранится:
- Название (например: "Ялян Маркет", "Барахолка точка 45")
- Контактное лицо
- Телефон
- Email
- Адрес / точка на рынке
- Заметки (условия работы, мин. заказ, способ оплаты)

## Когда товар уходит из списка

Товар **автоматически исчезает** из списка закупок, когда `stock_quantity` становится > `min_stock_level`.

Типичный сценарий:
1. Остаток Куклы Барби упал до 1 шт → появилась в списке
2. Вы видите: нужно дозаказать 10 шт у "Ялян Маркет"
3. Звоните поставщику (кнопка на странице), заказываете
4. Товар пришёл — обновляете остаток в карточке товара (1 + 10 = 11)
5. Кукла Барби исчезает из списка (11 > 2)

## Техническая реализация

### База данных

**Колонки в `products`**:
- `min_stock_level INT NOT NULL DEFAULT 2`
- `restock_quantity INT NOT NULL DEFAULT 5`
- `supplier_id UUID FK → suppliers.id` (ON DELETE SET NULL)

**RPC функция**: `get_restock_list()`
- Находит все активные товары где `stock_quantity ≤ min_stock_level`
- Группирует по поставщику (LEFT JOIN suppliers)
- Возвращает JSON с вложенной структурой: поставщик → товары
- Сортировка: по имени поставщика, внутри — по остатку (сначала самые критичные)

### Фронтенд

**Страница**: `/pages/admin/restock.vue`
- Вызывает `supabase.rpc('get_restock_list')`
- Десктоп: таблица с колонками (фото, товар, остаток, дозаказать, цена, поставщик)
- Мобайл: компактные карточки
- Inline-назначение поставщика для товаров без привязки
- Кнопка "Обновить" для ручного обновления списка
