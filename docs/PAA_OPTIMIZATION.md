### 1. База данных (Миграции)

#### Миграция 1: `20260331051200_add_priority_order_to_questions.sql`

**Добавлено поле `priority_order`:**

```sql
ALTER TABLE public.category_questions
ADD COLUMN priority_order INTEGER DEFAULT 999;
```

**Назначение:**

- Приоритет 1-10 = вопросы из Google PAA (вручную добавленные админом)
- Приоритет 100-400 = автогенерированные вопросы
- Приоритет 999 = обычные пользовательские вопросы

**Индекс для быстрой сортировки:**

```sql
CREATE INDEX idx_category_questions_priority_order
ON category_questions(category_id, priority_order, created_at);
```

**Автоматическая расстановка приоритетов:**

- "Что входит в категорию" → 100
- "Ценовой диапазон" → 200
- "Какие бренды" → 300
- "Доставка" → 400

---

#### Миграция 2: `20260331051201_optimize_category_questions_for_paa.sql`

**Полностью переписана функция `generate_category_questions()`**

**Ключевые изменения:**

1. **The Snippet Pattern** - Первое предложение = четкое определение (до 160 символов):

```sql
'<strong>Цены на ' || v_category_name || ' в Алматы варьируются от ' ||
replace(to_char(v_min_price::bigint, 'FM999,999,999'), ',', ' ') || ' ₸ до ' ||
replace(to_char(v_max_price::bigint, 'FM999,999,999'), ',', ' ') || ' ₸</strong>.'
```

2. **HTML-списки для перечислений:**

```sql
'У нас вы найдете:<ul>' ||
'<li>Бюджетные варианты для экономных покупателей</li>' ||
'<li>Премиум товары известных брендов</li>' ||
'<li>Акции и скидки на популярные модели</li>' ||
'</ul>'
```

3. **Внутренняя перелинковка:**

```sql
'<a href="https://uhti.kz/catalog/' || v_category_slug || '">Смотреть все цены на ' || v_category_name || '</a>.'
```

4. **Динамический список брендов:**

```sql
SELECT string_agg('<li>' || b.name || '</li>', '')
FROM (
  SELECT b.name, COUNT(p.id) as cnt
  FROM brands b
  INNER JOIN products p ON p.brand_id = b.id
  WHERE p.category_id = p_category_id
  GROUP BY b.id, b.name
  ORDER BY cnt DESC
  LIMIT 5
) b;
```

#### `components/category/CategoryQuestions.vue`

**Добавлена функция `sanitizeAndRenderHTML()`:**

```typescript
function sanitizeAndRenderHTML(html: string | null): string {
  if (!html) return "";

  // Разрешаем только безопасные теги для SEO
  const allowedTags = ["strong", "ul", "li", "a", "br", "p"];
  const tempDiv = document.createElement("div");
  tempDiv.innerHTML = html;

  // Удаляем все скрипты и опасные атрибуты
  const scripts = tempDiv.querySelectorAll("script, style");
  scripts.forEach((s) => s.remove());

  return tempDiv.innerHTML;
}
```

**Обновлен рендеринг ответов:**

```vue
<div
  class="text-sm leading-relaxed text-muted-foreground prose prose-sm max-w-none"
  v-html="sanitizeAndRenderHTML(q.answer_text)"
/>
```

**Добавлены стили для HTML-контента:**

```css
.prose :deep(strong) {
  @apply font-semibold text-foreground;
}

.prose :deep(ul) {
  @apply list-disc list-inside my-2 space-y-1;
}

.prose :deep(a) {
  @apply text-primary hover:underline font-medium;
}
```

**Результат:**

- Ключевые слова выделены жирным (`<strong>`)
- Списки отображаются корректно
- Ссылки кликабельны и стилизованы

#### `pages/catalog/[...slug].vue`

**Обновлена FAQPage Schema для чистого текста:**

```typescript
'text': (q.answer_text || 'Ответ скоро будет добавлен.')
  .replace(/<strong>/g, '')
  .replace(/<\/strong>/g, '')
  .replace(/<ul>/g, '\n')
  .replace(/<\/ul>/g, '')
  .replace(/<li>/g, '• ')
  .replace(/<\/li>/g, '\n')
  .replace(/<a[^>]*>/g, '')
  .replace(/<\/a>/g, '')
  .replace(/<br\s*\/?>/g, '\n')
  .replace(/<p>/g, '')
  .replace(/<\/p>/g, '\n')
  .replace(/\n\n+/g, '\n')
  .trim(),
```

**Почему это важно:**

- Google требует чистый текст в Schema.org
- HTML-теги конвертируются в читаемый формат
- Списки `<ul><li>` → маркированный список с `•`
- Сохраняется структура текста

## 🎯 Как это работает

### Жизненный цикл FAQ для категории

```
1. Админ создает/обновляет категорию
   ↓
2. Триггер вызывает generate_category_questions()
   ↓
3. Функция генерирует 4 базовых вопроса:
   - Что такое [категория]? (priority: 100)
   - Сколько стоят [категория]? (priority: 200)
   - Какие бренды [категория]? (priority: 300)
   - Как быстро доставите [категория]? (priority: 400)
   ↓
4. Ответы содержат:
   - Четкое определение в первом предложении
   - HTML-списки с перечислениями
   - Внутренние ссылки на каталог
   - Ключевые слова в <strong>
   ↓
5. Frontend:
   - Store загружает вопросы с сортировкой по priority_order
   - CategoryQuestions.vue рендерит HTML с безопасной санитизацией
   - FAQPage Schema конвертирует HTML в чистый текст
   ↓
6. Google индексирует:
   - Видит структурированные данные FAQPage
   - Извлекает четкие определения из первых предложений
   - Распознает списки (конвертированные в маркированный текст)
   - Использует uhti.kz как источник для PAA блока
```

## 🎨 Правила написания ответов для PAA

### 1. Первое предложение = Определение (до 160 символов)

**Плохо:**

```
В нашем магазине представлен широкий ассортимент бизибордов...
```

**Хорошо:**

```
<strong>Бизиборд — это развивающая игровая доска для детей от 6 месяцев до 5 лет, которая тренирует мелкую моторику и логическое мышление</strong>.
```

---

### 2. Используйте списки для перечислений

**Плохо:**

```
Бизиборд развивает мелкую моторику, логическое мышление, координацию движений и концентрацию внимания.
```

**Хорошо:**

```
Основные навыки:
<ul>
<li>Развитие мелкой моторики рук</li>
<li>Изучение причинно-следственных связей</li>
<li>Тренировка концентрации внимания</li>
<li>Освоение бытовых навыков</li>
</ul>
```

---

### 3. Добавляйте внутренние ссылки

**Формат:**

```html
<a href="https://uhti.kz/catalog/[slug]">Призыв к действию</a>
```

**Примеры:**

- "Смотреть все товары в категории Бизиборды"
- "Выбрать бизиборд для вашего ребенка"
- "Купить Бизиборды с доставкой"

---

### 4. Выделяйте ключевые слова

**Используйте `<strong>` для:**

- Определений
- Цифр и цен
- Важных фактов

**Пример:**

```html
<strong>Доставка Бизибордов по Алматы занимает 1-2 рабочих дня</strong>.
```

### 2. Проверка приоритетов

**SQL запрос:**

```sql
SELECT
  question_text,
  priority_order,
  is_auto_generated,
  LEFT(answer_text, 100) as answer_preview
FROM category_questions
WHERE category_id = 'uuid-категории'
ORDER BY priority_order ASC, created_at DESC;
```

**Ожидаемый результат:**

- Вопросы из Google PAA (priority 1-10) — сверху
