# Отчёт: Постоянство корзины и синхронизация с БД

## Дата: 2026-03-04

## Проблема

Корзина не синхронизировалась с сервером при логине/смене устройства:
1. При логине корзина НЕ отправлялась на сервер (watcher срабатывал только при изменении items)
2. При логине на новом устройстве серверная корзина НЕ загружалась
3. Отсутствовала merge-логика гостевой корзины с серверной при авторизации
4. Debounce 2000ms — слишком долгий для UX
5. Storage key `krakenshop-cart-v1` — устаревшее название

## Что было сделано

### 1. `stores/publicStore/cartStore.ts`

**a) Переименован storage key:**
- `krakenshop-cart-v1` → `uhti-cart-v1`

**b) Уменьшен debounce:**
- 2000ms → 500ms для более быстрой синхронизации

**c) Добавлен метод `forceSyncToServer()`:**
- Немедленная синхронизация без debounce
- Отменяет pending debounce перед выполнением
- Используется при merge на логине

**d) Добавлен метод `loadServerCart()`:**
- Загружает корзину из `server_carts` для текущего пользователя
- Для каждого product_id загружает полный продукт из `products` + `product_images`
- Возвращает массив `ICartItem[]`

**e) Добавлен метод `mergeOnLogin()`:**
- Вызывается при `SIGNED_IN` событии
- Логика merge:
  - Если локальная корзина НЕ пустая → `forceSyncToServer()` (локальная приоритетнее)
  - Если локальная корзина пустая → `loadServerCart()` и восстановление
- Безопасен для вызова из нескольких мест (идемпотентен)

**f) Добавлен метод `cancelPendingSync()`:**
- Отменяет pending debounce при logout
- Предотвращает отправку данных после выхода из аккаунта

### 2. `plugins/auth-init.client.ts`

- Импортирован `useCartStore`
- В обработчике `SIGNED_IN`: вызов `cartStore.mergeOnLogin()` после загрузки профиля
- В обработчике `SIGNED_OUT`: вызов `cartStore.cancelPendingSync()`

### 3. `app.vue`

- Импортирован `useCartStore`
- В watcher на `supabaseUser`:
  - При логине: `cartStore.mergeOnLogin()`
  - При логауте: `cartStore.cancelPendingSync()`

## Архитектура синхронизации

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   localStorage   │     │    cartStore      │     │  server_carts   │
│  (uhti-cart-v1) │◄────│  items: ref[]     │────►│  JSONB items    │
│                 │     │                  │     │                 │
│  persist plugin │     │  watcher → sync  │     │  abandoned cart │
│  auto-restore   │     │  debounce 500ms  │     │  cron job       │
└─────────────────┘     └──────────────────┘     └─────────────────┘
```

### Потоки данных

1. **Гость добавляет товар** → localStorage только
2. **Гость логинится (корзина не пустая)** → `forceSyncToServer()` → серверная корзина обновлена
3. **Пользователь логинится на новом устройстве** → `loadServerCart()` → localStorage заполнен
4. **Пользователь меняет количество** → watcher → debounced `syncToServer()` (500ms)
5. **Пользователь выходит** → `cancelPendingSync()` → localStorage НЕ очищается (гость продолжает)

## Тестирование

### Сценарии для ручной проверки:

1. **Persist через перезагрузку**: Добавить товар → перезагрузить страницу → корзина на месте
2. **Гость → логин**: Добавить товар как гость → залогиниться → корзина синхронизирована на сервер
3. **Новое устройство**: Залогиниться (пустой localStorage) → серверная корзина загружена
4. **Debounce**: Быстро кликать +/- → один запрос через 500ms
5. **Логаут**: Выйти → корзина остаётся в localStorage для гостя
